# スロット収支ノート — プロジェクトメモ

パチスロの収支を記録するWebアプリ。貯玉プレイ中心のユーザーが、**メダルの枚数で入力**し、**金額で収支**を把握するためのもの。

## 構成・方針（重要）

- **3ファイル構成**：`index.html`（マークアップ）/ `style.css`（スタイル）/ `app.js`（ロジック）。
  HTMLから相対パスで `style.css` と `app.js` を読み込む。`app.js` は `<body>` 末尾で読み込み、
  `#app` などのDOMが存在する前提（DOMContentLoaded待ちはしていない）。
  Reactや外部ライブラリは使わない。ビルド工程も持たない（素の3ファイルをそのまま配置）。
  Google Fontsのみ `style.css` 内の`@import`で読み込み（フォールバックあり）。
- **配布は URL（GitHub Pages）前提**。3ファイルを一緒に置くこと。ファイル単体を配るとスタイル/動作が欠ける。
- **データ保存はlocalStorage**（サーバーなし）。キーは `sessions`（記録の配列）と `lastRate`（最後に使った交換率）。
- **配布**：GitHub Pages。リポジトリ直下の `index.html` が公開URLのトップになり、同階層の `style.css` / `app.js` を読む。
  更新は変更したファイルを上書きコミットするだけ。
- スマホ最適化済み：viewport / セーフエリア対応 / ホーム画面追加メタ / 入力欄は16pxで自動ズーム防止。

## 収支計算ロジック（仕様の核。変更時は要注意）

交換率は「100円あたりの交換枚数（per100）」で持つ。等価=5枚（=1枚20円）。

- 換金単価（円/枚）= `100 / per100`
- `SAVE_UNIT = 20`（貯玉・貸玉は等価。1枚20円）

1記録の収支：
```
収支 = 換金枚数 × 換金単価
     + 貯玉に戻した枚数 × 20
     − 開始持ち玉（下ろした貯玉）× 20
     − 追加の現金投資
```

**考え方**：出玉のうち換金する分だけ交換率で評価し、貯玉に戻す分と元手の貯玉は「次回また等価で打てる資産」なので20円で評価する。これで現金プレイ・貯玉プレイ・その半々のいずれも同じ式で正しく出る。

旧バージョンのデータ（`endMedals` フィールドのみ）も読めるよう後方互換を残してある。`sessionProfit()` / `endTotal()` 内の分岐がそれ。**消さないこと。**

## データ構造（1記録）

```
{
  id,             // タイムスタンプ
  date,           // "YYYY-MM-DD"
  shop,           // 店名・機種（任意）
  memo,           // メモ（任意）
  expectedValue,  // 期待値 円（任意・メモとは別欄。収支計算には含めない参考値）
  startMedals,    // 開始持ち玉（下ろした貯玉）枚
  cashIn,         // 追加の現金投資 円
  cashOutMedals,  // 換金した枚数
  saveMedals,     // 貯玉に戻した枚数
  per100          // 交換率（100円あたりの枚数）
}
```

## 画面

- **カレンダータブ**：月表示。各日に収支を色付き（勝ち=緑/負け=赤）で表示。日をタップするとモーダルが開き、その日の記録の追加・編集・削除ができる。1日に複数記録可。今日を一発で開く「＋今日を記録」ボタンあり。見出しの「YYYY年M月」をタップすると年間の月選択モーダル（月別収支一覧）が開き、任意の月へ即移動できる。
- **推移タブ**：累計収支の折れ線グラフ、月別収支の棒グラフ、来店/勝ち/勝率/投資のサマリー。
- 交換率はプリセット（等価/5.6/5.8/6/6.5/7枚）＋カスタム入力。最後に使った率を記憶。
- ヒーロー部分（カレンダータブ上部）に累計収支・累計期待値を表示。期待値は記録ごとの入力値（`expectedValue`）を単純合計したもので、収支とは別軸の参考指標。
- データはJSONでエクスポート/インポート可能。

## 既知の前提

- 貯玉の再プレイは「等価」の店を前提に計算している。もし貯玉再プレイにも手数料/非等価レートがかかる店に対応するなら、`SAVE_UNIT` を可変にする改修が必要。
